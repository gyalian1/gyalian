2.2时间复杂度

1、使用步数作为时间复杂度的计算
数组的每次索引值得读取，就算作一步，称为unit_time
2、步数log（N），对数时间—O（log（N））
数组值获取（O（1））

3、猜数字游戏（O(logN))
O（log（N））比O（1）增长得更快一点，性能更差一点

时间复杂度为2N+2，用大O记法，线性时间为O（2N+2）

4、计算N的阶乘(O（N））
大O记法表示的是代码执行时间随数据规模增长的变化趋势，而在这种情况下，常量、系数部分并不左右增长趋势，所以可以忽略。
所以O（2 N+2）可以表示为O（N）

5、计算一个数组中的所有组合方式（O（N^2))

for (int j = i + 1; j < args.length; j++) { //step2

2.4性能优化小案例1-二分法查找

1、线性查找，依次遍历整个数组，判断给定的数字是否存在
public static int find(int[] array, int aim) {
for (int i = 0; i < array.length; i++) {
if (aim == array[i]) {
return i;
}
}
return -1;
}

2、二分查找法
定义两个变量left、right，分别表示数字可筛选区域左侧右侧的索引（下标）
public static int find(int[] array, int aim) {
// 初始化left = 最左侧, right = 最右侧
int left = 0;
int right = array.length - 1;

// 当left > right则表示遍历完成
while (left <= right) {
    // 获取中间的值
    int middle = (left + right) / 2;
    int middleValue = array[middle];
    if (middleValue == aim) {
        // 已经找到目标
        return middle;
    } else if (middleValue < aim) {
        // 目标在middle的右侧，重置left
        left = middle + 1;
    } else {
        // 目标在middle的左侧，重置right
        right = middle - 1;
    }
}
return -1;

}

2.5 性能优化小案例2–二次问题

1、暴力破解法
第一步，我们选择第一个元素 0, 判断与后面 7 个元素是否相同。 第二步，我们选择第二个元素 8, 判断与后面 6 个元素是否相同。 … 以此类推
代码：
public static ArrayList repeat(int[] array) {
ArrayList result = new ArrayList<>();
for(int i = 0; i < array.length; i++){
// 以此判断i位置元素和后面j位置元素是否相等
for(int j = i + 1; j < array.length; j++){
if(array[i] == array[j]){
result.add(array[i]);
}
}
}
return result;
}

2、标记法
首先申请一个数组，数组名称为exists，表示数组是否存在
exists[0] = 1;
exists[8] = 1;

public static ArrayList repeat(int[] array) {
ArrayList result = new ArrayList<>();
int[] exists = new int[11];
for (int i = 0; i < array.length; i++) {
int value = array[i];
// 如果当前位置已经为1，则表示重复
if (exists[value] == 1) {
result.add(value);
} else {
// 否则将当前位置设置为1
exists[value] = 1;
}
}
return result;
}
