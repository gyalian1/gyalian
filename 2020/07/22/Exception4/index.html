匿名类

比较重要
匿名类本质上就是一个局部内部类，不同的是匿名类只用于类的实例化，而不是声明

有一个接口A
public interface A {
public void method();
}

public class ATest {
public static void main(String[] args){
A a = new A(){
public void method(){
System.out.println(“执行内部类”);
}
};
a.method();
}
}

匿名内除了实现接口，也可以继承父类
public class Sub {
public void print(){
}
}
public class SubTest {
public static void main(String[] args){
Sub sub = new Sub(){
public void print(){
System.out.println(“执行内部类”);
}
};
sub.print();
}
}

5.3接口中的默认方法

abstract 需要声明在 class 类上以及需要抽象的方法上
// 定义一个武器的抽象类
public abstract class Weapon{
// 抽象方法：攻击
public abstract void attack();
}

他的子类必须也实现抽象方法
public class GoldenBar extends Weapon {
public void attack(){
System.out.println(“使用金箍棒进行劈一棍”);
}
}

接口的默认方法
public interface Vehicle {
default void print(){
System.out.println(“我是一辆车!”);
}
}

我们在接口的方法中使用了 default关键字，这个时候实现类如果没有自己的实现逻辑，就可以不用覆盖默认方法

覆盖接口默认方法实现
public class Car implements Vehicle {
public void print(){
System.out.println(“我是一辆汽车!”);
}
}

非覆盖接口默认方法实现
public class Car implements Vehicle {
}

public class CarTester {
public static void main(String args[]){
Vehicle vehicle = new Car();
vehicle.print();
}
}
